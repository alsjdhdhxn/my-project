# 1.7 自定义导出（配置说明）

## 配置入口
- **T_COST_EXPORT_CONFIG**（主表导出配置）
- **T_COST_EXPORT_CONFIG_DETAIL**（从表配置）

## 导出效果

主表所有记录在一个Sheet，每条主表记录的从表数据生成一个独立Sheet。

**示例：主表有3条记录（ID=1,2,3），每条记录有原辅料和包材两个从表**

## 核心字段说明

### T_COST_EXPORT_CONFIG（主表配置）
| 字段 | 必填 | 说明 |
|------|------|------|
| EXPORT_CODE | 是 | 导出标识（唯一） |
| EXPORT_NAME | 是 | 导出名称（显示在右键菜单） |
| PAGE_CODE | 是 | 绑定的页面编码 |
| MASTER_SQL | 是 | 主表查询SQL |
| MASTER_TABLE_ALIAS | 是 | 主表SQL别名 |
| PK_COLUMN | 是 | 主键列 |
| PAGE_FK_COLUMN | 是 | 页面视图关联列 |
| MASTER_SHEET_NAME | 否 | 主表Sheet名称，默认"master" |
| COLUMNS | 否 | 列配置JSON，不填则自动推断 |
| DISPLAY_ORDER | 否 | 显示顺序 |

### T_COST_EXPORT_CONFIG_DETAIL（从表配置）
| 字段 | 必填 | 说明 |
|------|------|------|
| EXPORT_CONFIG_ID | 是 | 关联主表配置ID |
| TAB_KEY | 否 | 对应前端tabKey |
| SHEET_NAME | 否 | 数据块标题（显示在数据上方，如"原辅料明细"） |
| DETAIL_SQL | 是 | 从表查询SQL |
| DETAIL_LINK_COLUMN | 是 | 从表关联主表的列名（如"MASTER_ID"） |
| COLUMNS | 否 | 列配置JSON，不填则自动推断 |
| DISPLAY_ORDER | 否 | 显示顺序（决定从表在Sheet中的排列顺序） |

---

## 列配置 JSON 格式
```json
[
  {"field": "NAME", "header": "名称"},
  {"field": "AMOUNT", "header": "金额"},
  {"field": "REMARK", "header": "备注", "visible": false}
]
```
- `visible: false` 表示导出时跳过该列

## 配置示例

```sql
DECLARE
    v_config_id NUMBER;
BEGIN
    SELECT SEQ_COST_EXPORT_CONFIG.NEXTVAL INTO v_config_id FROM DUAL;
    
    -- 主表配置
    INSERT INTO T_COST_EXPORT_CONFIG (
        ID, EXPORT_CODE, EXPORT_NAME, PAGE_CODE,
        MASTER_SQL, MASTER_TABLE_ALIAS, PK_COLUMN, PAGE_FK_COLUMN,
        MASTER_SHEET_NAME, COLUMNS, DISPLAY_ORDER, CREATE_BY
    ) VALUES (
        v_config_id,
        'cost-pinggu-export',
        '物料成本核算表导出',
        'cost-pinggu',
        'SELECT DOCID, GOODSNAME, STRENGTH, CUSTOMNAME FROM T_COST_PINGGU WHERE DELETED = 0',
        'm',
        'DOCID',
        'DOCID',
        '物料成本核算',
        '[{"field":"DOCID","header":"ID"},{"field":"GOODSNAME","header":"产品名称"},{"field":"STRENGTH","header":"剂量"},{"field":"CUSTOMNAME","header":"客户名称"}]',
        1,
        'system'
    );

    -- 从表1：原辅料明细
    INSERT INTO T_COST_EXPORT_CONFIG_DETAIL (
        ID, EXPORT_CONFIG_ID, TAB_KEY, SHEET_NAME, 
        DETAIL_SQL, DETAIL_LINK_COLUMN, COLUMNS, DISPLAY_ORDER
    ) VALUES (
        SEQ_COST_EXPORT_CONFIG_DTL.NEXTVAL,
        v_config_id,
        'material',
        '原辅料明细',
        'SELECT MASTER_ID, DTL_USEFLAG, APEX_GOODSNAME, SPEC, PRICE, BATCH_QTY, COST_BATCH FROM V_COST_PINGGU_MATERIAL WHERE DELETED = 0',
        'MASTER_ID',
        '[{"field":"DTL_USEFLAG","header":"分类"},{"field":"APEX_GOODSNAME","header":"物料名称"},{"field":"SPEC","header":"规格"},{"field":"PRICE","header":"单价"},{"field":"BATCH_QTY","header":"每批数量"},{"field":"COST_BATCH","header":"每批成本"}]',
        1
    );

    -- 从表2：包材明细
    INSERT INTO T_COST_EXPORT_CONFIG_DETAIL (
        ID, EXPORT_CONFIG_ID, TAB_KEY, SHEET_NAME, 
        DETAIL_SQL, DETAIL_LINK_COLUMN, COLUMNS, DISPLAY_ORDER
    ) VALUES (
        SEQ_COST_EXPORT_CONFIG_DTL.NEXTVAL,
        v_config_id,
        'package',
        '包材明细',
        'SELECT MASTER_ID, DTL_USEFLAG, APEX_GOODSNAME, SPEC, PRICE, BATCH_QTY, COST_BATCH FROM V_COST_PINGGU_PACKAGE WHERE DELETED = 0',
        'MASTER_ID',
        '[{"field":"DTL_USEFLAG","header":"包材类型"},{"field":"APEX_GOODSNAME","header":"包材名称"},{"field":"SPEC","header":"规格"},{"field":"PRICE","header":"单价"},{"field":"BATCH_QTY","header":"每批数量"},{"field":"COST_BATCH","header":"包材成本"}]',
        2
    );

    COMMIT;
END;
/
```

## 导出模式
- **导出全部**：不带筛选条件，导出所有数据
- **导出当前**：携带当前表格的筛选/排序条件

---

## 注意事项
1. 前端右键菜单会自动显示"自定义导出"选项
2. **从表必须配置 `DETAIL_LINK_COLUMN`**，指定关联主表的列名
3. 从表SQL必须查询出 `DETAIL_LINK_COLUMN` 指定的列
4. `COLUMNS` JSON 未配置时，会按数据列自动推断
5. Sheet名称最长31个字符，超出会自动截断
6. 从表的 `SHEET_NAME` 会作为数据块标题显示在数据上方
