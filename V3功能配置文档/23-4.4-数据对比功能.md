# 4.4 数据对比功能

## 功能概述

数据对比功能允许在表格中显示当前值与对比值的差异，支持显示差值、百分比变化，并用颜色标识涨跌趋势。

## 配置位置

在列元数据的 `RULES_CONFIG` 字段中配置 `compare` 对象。

## 配置模式

### 1. 视图字段模式（viewField）

对比字段已在当前视图中（通过 SQL VIEW 预先 JOIN 好），性能最优。

```json
{
  "compare": {
    "enabled": true,
    "mode": "viewField",
    "compareField": "HIS_PRICE",
    "format": "percent",
    "upColor": "#e53935",
    "downColor": "#43a047"
  }
}
```

**字段说明：**
- `enabled`: 是否启用对比功能
- `mode`: 固定为 `"viewField"`
- `compareField`: 对比字段名（同一数据源中的字段）
- `format`: 显示格式
  - `"value"`: 仅显示数值差
  - `"percent"`: 仅显示百分比
  - `"both"`: 两者都显示
- `upColor`: 上涨颜色（默认 `#e53935` 红色）
- `downColor`: 下跌颜色（默认 `#43a047` 绿色）

**适用场景：**
- 视图中已通过 LEFT JOIN 关联好历史数据
- 对比字段与当前字段在同一查询结果中

### 2. 动态查询模式（dynamicQuery）

运行时动态查询对比数据源，灵活但有额外查询开销。

```json
{
  "compare": {
    "enabled": true,
    "mode": "dynamicQuery",
    "compareDataSource": "MV_COST_PINGGU_DTL_HIS_LATEST",
    "joinConditions": [
      { "currentField": "id", "compareField": "DTLID" }
    ],
    "compareField": "PRICE",
    "format": "both",
    "upColor": "#e53935",
    "downColor": "#43a047"
  }
}
```

**字段说明：**
- `mode`: 固定为 `"dynamicQuery"`
- `compareDataSource`: 对比数据源（视图或表名）
- `joinConditions`: 关联条件数组，支持多字段关联
  - `currentField`: 当前数据的字段名（驼峰格式）
  - `compareField`: 对比数据源的字段名（数据库列名）
- `compareField`: 对比数据源中的对比值字段

**适用场景：**
- 对比数据在独立的视图/表中
- 需要灵活配置关联条件
- 不想修改主查询视图

## 前端渲染效果

配置生效后，单元格会显示：
- 当前值
- 涨跌箭头（↑ 或 ↓）
- 差值/百分比（根据 format 配置）
- 颜色标识（根据 upColor/downColor 配置）

示例：`123.45 ↑5.2%`

## 后端返回数据

后端会自动在返回数据中添加以下字段：
- `{fieldName}Compare`: 对比值
- `{fieldName}Diff`: 差值（当前值 - 对比值）
- `{fieldName}DiffPercent`: 百分比变化

## 完整配置示例

### 示例1：视图字段模式

```sql
-- 列元数据配置
UPDATE T_COST_COLUMN_METADATA 
SET RULES_CONFIG = '{"compare":{"enabled":true,"mode":"viewField","compareField":"hisCostBatch","format":"percent"}}'
WHERE TABLE_CODE = 'COST_PINGGU_DTL' AND FIELD_NAME = 'costBatch';
```

### 示例2：动态查询模式

```sql
-- 列元数据配置
UPDATE T_COST_COLUMN_METADATA 
SET RULES_CONFIG = '{"compare":{"enabled":true,"mode":"dynamicQuery","compareDataSource":"MV_COST_PINGGU_DTL_HIS_LATEST","joinConditions":[{"currentField":"id","compareField":"DTLID"}],"compareField":"COST_BATCH","format":"both","upColor":"#e53935","downColor":"#43a047"}}'
WHERE TABLE_CODE = 'COST_PINGGU_DTL' AND FIELD_NAME = 'costBatch';
```

## 注意事项

1. **性能考虑**：优先使用视图字段模式，动态查询模式会产生额外的 SQL 查询
2. **字段命名**：
   - `currentField` 使用驼峰格式（如 `id`）
   - `compareField` 使用数据库列名格式（如 `DTLID`）
3. **多字段关联**：`joinConditions` 支持多个条件，用于复合主键场景
4. **颜色配置**：默认红涨绿跌，可根据业务需求调整（如成本场景可能需要绿涨红跌）
