# 2.5 执行器服务（Action）配置说明

## 配置入口
执行器支持两种配置方式：

1. **T_COST_TABLE_METADATA.ACTION_RULES**（JSON 数组）- 表级别配置
2. **T_COST_PAGE_RULE** 的 `CONTEXT_MENU` / `TOOLBAR` 规则 - 页面级别内联配置

## 规则结构（ActionRule）
```json
{
  "order": 1,
  "code": "APPROVE",
  "name": "审核",
  "group": "toolbar",
  "enabled": true,
  "type": "sql",          
  "sql": "UPDATE T_COST_MASTER SET STATUS = '已审核' WHERE ID = ${id}",
  "description": "主表审核"
}
```

### 支持的 type
- `sql`：执行 SQL
- `java`：执行 Spring Bean 方法（method = beanName.methodName）
- `proc`：执行存储过程（procedure + params）

## 方式一：表级别配置（ACTION_RULES）

适用于多个页面共用的通用操作。

### 示例 A：SQL 执行器
```sql
UPDATE T_COST_TABLE_METADATA
SET ACTION_RULES = '[
  {"order":1,"code":"APPROVE","group":"toolbar","enabled":true,
   "type":"sql","sql":"UPDATE T_COST_MASTER SET STATUS = ''已审核'' WHERE ID = ${id}"}
]'
WHERE TABLE_CODE = 'CostMaster';
```

### 示例 B：Java 执行器
```json
[
  {"order":1,"code":"SYNC","type":"java","method":"costActionHandler.sync","enabled":true}
]
```

### 示例 C：存储过程执行器
```json
[
  {
    "order":1,
    "code":"CALC_PROC",
    "type":"proc",
    "procedure":"PKG_COST.CALC_COST",
    "params":[
      {"name":"P_ID","mode":"IN","jdbcType":"NUMERIC","source":"id"},
      {"name":"P_MSG","mode":"OUT","jdbcType":"VARCHAR","target":"msg"}
    ]
  }
]
```

## 方式二：页面级别内联配置（推荐）

直接在 `CONTEXT_MENU` 或 `TOOLBAR` 规则中配置执行器，无需在 ACTION_RULES 中预定义。

### 右键菜单内联存储过程
```sql
INSERT INTO T_COST_PAGE_RULE (ID, PAGE_CODE, COMPONENT_KEY, RULE_TYPE, RULES, CREATE_BY)
VALUES (
  SEQ_COST_PAGE_RULE.NEXTVAL,
  'cost-pinggu',
  'masterGrid',
  'CONTEXT_MENU',
  '{"items":[
    {"action":"addRow"},
    {"action":"generateFromBom","label":"由BOM生成明细","requiresRow":true,
     "procedure":"P_COST_BOM_INSERT","params":[{"source":"data.docid","mode":"IN","jdbcType":"NUMERIC"}]},
    {"action":"save"}
  ]}',
  'system'
);
```

### 右键菜单内联 SQL
```json
{
  "action": "approve",
  "label": "审核",
  "requiresRow": true,
  "sql": "UPDATE T_COST_MASTER SET STATUS = '已审核' WHERE ID = ${id}"
}
```

### params 参数说明
| 字段 | 说明 | 示例 |
|------|------|------|
| source | 参数来源，`data.字段名` 从当前行取值 | `data.docid` |
| target | OUT 参数输出目标 | `vars.result` |
| mode | 参数模式：IN / OUT / INOUT，默认 IN | `IN` |
| jdbcType | JDBC 类型，默认 VARCHAR | `NUMERIC`, `VARCHAR` |

## 触发方式
- 前端按钮：componentConfig.action = "APPROVE" + tableCode
- 右键菜单：点击菜单项自动触发
- 接口调用：POST `/api/data/{tableCode}/execute`
```json
{"group":"toolbar","actionCodes":["APPROVE"],"data":{"id":1001},"validate":true}
```

## 注意事项
- 若同时传 group 与 actionCodes，会优先按 actionCodes 过滤。
- validate=true 会先走后端验证（validationRules）。
- 内联配置的执行器执行成功后会自动刷新表格数据。
