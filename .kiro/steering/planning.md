# 文件式计划管理（Planning with Files）

基于 Manus AI 的上下文工程模式：将 AI 的"工作记忆"持久化到文件，而非依赖上下文窗口。

## 核心原则

- 上下文窗口 = 内存（易失、有限）
- 文件系统 = 硬盘（持久、无限）
- 重要信息必须写入文件

## 何时启用

适用于：
- 多步骤任务（3 步以上）
- 研究型任务
- 跨多文件的构建/重构
- 需要大量工具调用的任务

不适用于：
- 简单问答
- 单文件修改
- 快速查询

## 三文件模式

每次复杂任务开始前，在项目根目录创建三个文件：

| 文件 | 用途 | 更新时机 |
|------|------|----------|
| `task_plan.md` | 阶段划分、进度追踪、决策记录 | 每个阶段完成后 |
| `findings.md` | 研究发现、中间结果 | 每次有新发现时 |
| `progress.md` | 会话日志、测试结果 | 全程持续更新 |

## 关键规则

1. 先建计划：复杂任务必须先创建 `task_plan.md`，不可跳过
2. 二步保存规则：每执行 2 次查看/搜索操作后，立即将关键发现写入文件
3. 决策前回读：做重大决策前，重新读取计划文件，刷新目标
4. 行动后更新：完成任何阶段后，标记状态、记录错误、记录修改的文件
5. 记录所有错误：每个错误都写入计划文件，避免重复犯错
6. 不重复失败：如果某个方法失败了，下次必须换方法

## 错误处理协议

- 第 1 次：诊断根因，针对性修复
- 第 2 次：换方法，用不同工具或不同思路
- 第 3 次：重新审视假设，搜索解决方案
- 3 次失败后：向用户说明尝试过的方法和具体错误，请求指导

## 计划文件模板

### task_plan.md
```markdown
# 任务：[任务描述]

## 目标
[明确的目标描述]

## 阶段
- [ ] 阶段 1：[描述]
- [ ] 阶段 2：[描述]
- [ ] 阶段 3：[描述]

## 决策记录
| 决策 | 原因 | 时间 |
|------|------|------|

## 错误记录
| 错误 | 尝试次数 | 解决方案 |
|------|----------|----------|
```

### findings.md
```markdown
# 研究发现

## 关键发现
- [发现 1]

## 代码分析
- [分析结果]
```

### progress.md
```markdown
# 进度日志

## 当前状态：[进行中/已完成]

## 会话记录
### [日期]
- [操作和结果]

## 修改的文件
- [文件路径]: [修改说明]
```
