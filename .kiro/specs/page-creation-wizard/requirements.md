# 页面创建向导 — 需求文档

## 简介

页面创建向导是元数据驱动低代码平台的核心提效功能。当前创建一个新业务页面需要管理员手动执行 **10+ 步操作**。本向导将流程简化为 **3 步**，实现一键式页面搭建。

## 流程总览

```
┌─────────────────────────────────────────────────────────────┐
│                    页面创建向导 (3 步)                         │
├─────────────┬─────────────────────┬─────────────────────────┤
│  第1步       │  第2步               │  第3步                   │
│  创建目录    │  导入表与配置字段     │  确认生成                │
│             │                     │                         │
│  ○ 选父级   │  ○ 选模式(单表/主从)  │  ○ 查看生成清单          │
│  ○ 填名称   │  ○ 填视图名/表名      │  ○ 确认无误              │
│  ○ 改编码   │  ○ 查看/调整字段      │  ○ 一键提交              │
│             │  ○ 配置主从关系       │                         │
└─────────────┴─────────────────────┴─────────────────────────┘
```

## 术语表

| 术语 | 说明 |
|------|------|
| 向导 (Wizard) | 引导用户 3 步完成新页面配置的界面 |
| 目录 (Resource) | T_COST_RESOURCE 中的菜单/目录节点 |
| 表元数据 (Table_Metadata) | T_COST_TABLE_METADATA 中的表定义 |
| 列元数据 (Column_Metadata) | T_COST_COLUMN_METADATA 中的列定义 |
| 页面组件 (Page_Component) | T_COST_PAGE_COMPONENT 中的组件定义 |
| 页面规则 (Page_Rule) | T_COST_PAGE_RULE 中的规则配置 |
| 列覆盖 (COLUMN_OVERRIDE) | 控制列可见性、可编辑性的规则 |
| 真实列 (Real_Column) | 视图和目标表中都存在的列，可编辑可保存 |
| 虚拟列 (Virtual_Column) | 只在视图中存在的列，只读不保存 |
| 单表模式 | 只配置一张主表 |
| 主从模式 | 配置一张主表 + 一或多张从表 |
| 生成清单 (Manifest) | 第3步展示的待创建实体列表 |
| 向导载荷 (Payload) | 前端提交给后端的完整数据 |

## 需求 1：向导入口与步骤导航

**用户故事：** 作为管理员，我希望通过一个分步向导界面来创建页面，避免在多个配置面板之间来回跳转。

```
┌──────────────────────────────────────────────┐
│  [①创建目录] ──── [②导入表与配置字段] ──── [③确认生成]  │
│       ●                  ○                    ○       │
│                                                      │
│  ┌─────────────────────────────────────────┐         │
│  │          当前步骤的内容区域               │         │
│  │                                         │         │
│  └─────────────────────────────────────────┘         │
│                                                      │
│            [上一步]              [下一步]              │
└──────────────────────────────────────────────┘
```

#### 验收标准

1. 向导作为元数据配置中心的**新 Tab 页**呈现
2. 顶部显示 3 步进度条，当前步骤高亮
3. 第1步必填项未填满时，"下一步"按钮**禁用**
4. 第2步至少有一张表且有列配置后，"下一步"才可点
5. 点击"上一步"时，**已填数据不丢失**
6. 必填项为空时点击"下一步"，**高亮提示**缺失字段

## 需求 2：创建目录（第1步）

**用户故事：** 作为管理员，我希望选择父级目录并定义新目录，让生成的页面出现在正确的菜单位置。

```
第1步界面示意：

┌─────────────────────────────────────────────┐
│  父级目录：[树形选择器 ▾]                     │
│                                             │
│  目录名称：[____物料成本核算____]              │
│                                             │
│  目录编码：[COST_PINGGU________] (可手动修改) │
│                                             │
│  图标：    [folder ▾]            (可手动选择) │
└─────────────────────────────────────────────┘
```

#### 验收标准

1. 加载时显示 T_COST_RESOURCE 的**树形菜单**供选择父级
2. 选中父级后，记录其 ID
3. 输入目录名称后，**自动生成**编码（UPPER_SNAKE_CASE，最长 64 字符）
4. 编码允许手动修改，限定大写字母+数字+下划线
5. 自动分配默认图标 "folder"，允许手动更改
6. 名称为空时提示"目录名称必填"
7. 未选父级时提示"请选择父级目录"
8. 编码重复时提示"编码已存在"
9. 名称超 128 字符时提示"超出长度限制"

## 需求 3：表模式选择（第2步-上半部分）

**用户故事：** 作为管理员，我希望在单表和主从模式之间切换，让向导适应不同的页面结构。

```
┌─────────────────────────────────────────────────┐
│  页面模式：  (●) 单表    ( ) 主从                  │
├─────────────────────────────────────────────────┤
│                                                 │
│  【单表模式】                                    │
│  查询视图名：[V_COST_PINGGU_______]              │
│  目标表名：  [T_COST_PINGGU_______]              │
│                                                 │
│  ─── 或 ───                                     │
│                                                 │
│  【主从模式】                                    │
│  主表视图：  [V_COST_PINGGU_______]              │
│  主表目标：  [T_COST_PINGGU_______]              │
│                                                 │
│  从表1 视图：[V_COST_PINGGU_MATERIAL]  [× 删除]  │
│  从表1 目标：[T_COST_PINGGU_DTL______]           │
│                                                 │
│  从表2 视图：[V_COST_PINGGU_PACKAGE_]  [× 删除]  │
│  从表2 目标：[T_COST_PINGGU_DTL______]           │
│                                                 │
│  [+ 添加从表]                                    │
└─────────────────────────────────────────────────┘
```

#### 验收标准

1. 显示模式选择器：单表 / 主从
2. 单表模式显示 1 组视图+目标表输入框（各最长 64 字符）
3. 主从模式显示主表 + 至少 1 组从表输入框
4. 主从模式允许添加从表，**最多 5 张**；超过 5 张时显示警告"从表数量较多，建议拆分为多个页面"
5. 最后一组从表不可删除（至少保留 1 张）
6. 切换模式时**清空**之前填写的值
7. 视图名/表名只允许字母、数字、下划线

## 需求 4：从 Oracle 视图导入列

**用户故事：** 作为管理员，我希望系统自动读取 Oracle 视图的列定义，不需要手动逐列录入。

```
导入逻辑示意（简单列名匹配）：

   视图 V_COST_PINGGU             目标表 T_COST_PINGGU
   ┌──────────────────┐          ┌──────────────────┐
   │ DOCID            │◄─────────│ DOCID            │  ✓ 真实列
   │ GOODSNAME        │◄─────────│ GOODSNAME        │  ✓ 真实列
   │ CUSTOMNAME       │◄─────────│ CUSTOMNAME       │  ✓ 真实列
   │ FACTORYNAME      │          │                  │  ✗ 虚拟列（目标表无此列）
   │ GOODSTYPE        │          │                  │  ✗ 虚拟列（目标表无此列）
   └──────────────────┘          └──────────────────┘

   规则：视图列名 与 目标表列名 做不区分大小写的直接匹配。
   匹配 → 真实列（TARGET_COLUMN = 匹配到的目标表列名）
   不匹配 → 虚拟列（TARGET_COLUMN 为空，用户可手动指定）
```

#### 验收标准

1. 填写视图名后，通过 DBA_TAB_COLUMNS 查询视图所有列（按 COLUMN_ID 排序）
2. 填写目标表名后，查询目标表所有列
3. 视图列名与目标表列名做**不区分大小写的直接匹配**：匹配→真实列，不匹配→虚拟列
4. 真实列默认：IS_VIRTUAL=0，可编辑=是，参与保存=是，TARGET_COLUMN = 匹配到的目标表列名
5. 虚拟列默认：IS_VIRTUAL=1，可编辑=否，参与保存=否，TARGET_COLUMN 为空
6. 视图不存在时提示"视图未找到"
7. 目标表不存在时提示"目标表未找到"
8. 视图无列时提示"未找到列信息"
9. 导入成功后自动填充：字段名（COLUMN_NAME）、TARGET_COLUMN、标题（取 COMMENTS，无则用列名）、数据类型（NUMBER→number, DATE→date, 其他→text）、排序号（按 COLUMN_ID 递增）

## 需求 5：主从关系外键配置

**用户故事：** 作为管理员，我需要明确指定主从表的关联字段，确保主从关系正确。

```
主从关联配置：

   主表 CostPinggu            从表 CostMaterial
   ┌──────────────┐          ┌──────────────────┐
   │ PK: DOCID    │──────────│ FK: [用户选择 ▾]   │
   └──────────────┘          │   DOCID           │
                             │   GOODSID         │
                             │   DTLID           │
                             └──────────────────┘

   用户必须手动确认/选择从表的关联字段。
   系统可以预选（如果从表有与主表 PK 同名的列），但不强制。
```

#### 验收标准

1. 主从模式下，系统展示主表 PK 字段（从数据库约束中获取），并提供从表列的下拉列表让用户**手动选择** FK 列
2. 如果从表存在与主表 PK 同名的列，系统**预选**该列作为建议值，但用户可以修改
3. 用户**必须确认**关联字段后才能进入下一步
4. 如果用户未选择 FK 列就点击下一步，显示"请选择从表关联字段"的验证错误

## 需求 6：字段配置表格

**用户故事：** 作为管理员，我希望在一个表格中查看和微调导入的字段属性。

```
┌──────┬──────────┬──────┬────────┬────────────┬────┬────┬────┬──────┬────┐
│字段名 │ 标题     │类型  │真实/虚拟│TARGET_COL  │显示│编辑│查询│控件   │排序│
├──────┼──────────┼──────┼────────┼────────────┼────┼────┼────┼──────┼────┤
│DOCID │ ID       │number│ 真实   │DOCID       │ ✓  │ ✗  │ ✗  │number│ 1  │
│NAME  │ 产品名称 │text  │ 真实   │GOODS_NAME  │ ✓  │ ✓  │ ✓  │text  │ 2  │
│TYPE  │ 分类     │text  │ 虚拟   │            │ ✓  │ ✗  │ ✓  │select│ 3  │
│PRICE │ 单价     │number│ 真实   │PRICE       │ ✓  │ ✓  │ ✗  │number│ 4  │
│...   │          │      │        │            │    │    │    │      │    │
└──────┴──────────┴──────┴────────┴────────────┴────┴────┴────┴──────┴────┘
```

#### 验收标准

1. 显示可编辑表格，包含：字段名、标题、数据类型、真实/虚拟、TARGET_COLUMN、是否显示、是否可编辑、是否查询、控件类型、排序号
2. 允许编辑：标题（≤128字符）、数据类型（text/number/date）、显示、编辑、查询、控件类型（text/number/date/select/checkbox）、排序号（1-999）
3. "真实/虚拟"列**不可直接编辑**（由系统对比结果决定）
4. 标题优先用 Oracle COMMENTS，没有则用列名
5. 数据类型按 Oracle DATA_TYPE 自动映射
6. 排序号重复时**高亮提示**，阻止进入下一步
7. **虚拟列编辑校验**：如果用户将虚拟列的"可编辑"改为是，系统必须校验该列的 TARGET_COLUMN 是否在目标表中存在；如不存在则阻止并提示"目标表中不存在对应保存字段，无法设为可编辑"
8. 虚拟列可以由用户手动指定 TARGET_COLUMN，指定后如果目标表存在该列则**自动转为真实列**

## 需求 8：后端事务性生成

**用户故事：** 作为管理员，我希望所有配置在一个事务中保存，任何一步失败都整体回滚。

```
后端处理流程：

  接收 Wizard_Payload
        │
        ▼
  ┌─ BEGIN TRANSACTION ─────────────────────────────┐
  │  1. 创建 Resource (目录)                          │
  │  2. 创建 Table_Metadata (表元数据)                │
  │  3. 创建 Column_Metadata (列元数据)               │
  │  4. 创建 Page_Component (页面组件)                │
  │  5. 创建 Page_Rule (COLUMN_OVERRIDE + BUTTON)    │
  │                                                 │
  │  任何一步失败 → ROLLBACK 全部                     │
  └─────────────────────────────────────────────────┘
        │
        ▼ 成功
  清缓存 + WebSocket 广播 META_CONFIG_CHANGED
        │
        ▼
  返回 pageCode 给前端
```

#### 验收标准

1. 按依赖顺序创建：Resource → Table_Metadata → Column_Metadata → Page_Component → Page_Rule
2. 任何步骤失败 → **全部回滚**，返回错误（含失败类型和原因）
3. PAGE_CODE 已存在 → 拒绝，返回"页面编码重复"
4. 成功后清除元数据缓存 + 广播 WebSocket
5. 成功后返回 PAGE_CODE
6. 复用已有 MetaConfigService 方法

## 需求 9：自动生成默认值

**用户故事：** 作为管理员，我希望系统自动填充按钮、序列名、主键等默认值，让页面开箱即用。

#### 验收标准

1. 单表模式默认按钮：新增、删除、保存、查询
2. 主从模式默认按钮：主表（查询/新增/删除/保存）+ 每张从表（新增/删除/保存）
3. PAGE_CODE 自动推导：从视图名去掉 "V_" 前缀转 kebab-case，或从目录编码转 kebab-case
4. SEQUENCE_NAME = "SEQ_" + TARGET_TABLE
5. PK_COLUMN **必须从数据库约束中获取**（查询 ALL_CONSTRAINTS + ALL_CONS_COLUMNS 获取主键列）
6. 如果目标表在数据库中**没有主键约束**，则报错"目标表缺少主键定义，请先在数据库中创建主键"
7. COMPONENT_KEY 直接用 REF_TABLE_CODE 的值

## 需求 10：生成后跳转预览

**用户故事：** 作为管理员，我希望生成完成后直接跳转到新页面，立即看到效果。

#### 验收标准

1. 生成成功后，前端**刷新资源菜单、路由表和元数据缓存**
2. 刷新完成后自动跳转到新页面路由
3. 新页面正常渲染：列定义、按钮、组件布局
4. 跳转失败 → 显示"生成成功"+ 手动链接供用户点击跳转

## 需求 11：COLUMN_OVERRIDE 规则自动生成

**用户故事：** 作为管理员，我希望向导根据字段配置自动生成列覆盖规则，让可见性和可编辑性立即生效。

#### 验收标准

1. 为每个 COMPONENT_KEY 生成一条 COLUMN_OVERRIDE 规则（JSON 数组，每列一个对象）
2. 字段标记为显示 → `visible: true`
3. 字段标记为不显示 → `visible: false`
4. 真实列标记为可编辑 → `editable: true`
5. **提交时仍为虚拟列的字段**（IS_VIRTUAL=1）→ 强制 `editable: false`（即使用户勾选了编辑也忽略）
6. 如果用户为虚拟列指定了 TARGET_COLUMN 且校验通过（目标表存在该列），则该字段**已转为真实列**，按真实列规则处理（editable 取用户配置值）
6. 根据数据类型/控件类型自动设置 cellEditor 和 width（60-600px）
7. 无列配置 → 报错"列元数据缺失"
8. 同 PAGE_CODE + COMPONENT_KEY 已有规则 → 覆盖替换

---

## 需求 12：权限控制

**用户故事：** 作为系统，我需要限制只有管理员才能使用向导，防止未授权用户创建页面。

#### 验收标准

1. 后端校验请求用户为超级管理员（username = "admin"）
2. 非管理员访问向导接口 → 返回 403
3. 未登录访问 → 返回 401
4. 前端非管理员用户 → **不渲染**向导 Tab
