# 元数据配置中心一体化改造方案

## 目标

将现有的 6 个独立 Tab（目录管理、表管理、页面管理、Lookup 管理、导出配置、审批流配置）整合为**左树右面板**的一体化配置界面。用户选中一个页面节点后，在右侧面板内完成该页面的所有配置，不再需要跨 Tab 跳转。

## 现状问题

1. 配置一个页面需要在 4-5 个 Tab 之间来回切换
2. Tab 之间没有上下文关联，用户需要记住 pageCode、componentKey 等标识符
3. 无法一屏看到一个页面的完整配置状态
4. 目录管理中看不到每个页面配了多少规则、配置是否完整

## 设计原则

- **操作一体化**：所有配置在一个界面内完成
- **不损失功能**：现有所有编辑能力全部保留
- **简洁但不隐藏**：折叠面板展示摘要，展开后完整编辑

### 左侧目录树

- 展示 T_COST_RESOURCE 的树形结构（DIRECTORY + PAGE）
- 支持搜索过滤（按名称/pageCode）
- 支持拖拽调整排序和层级
- DIRECTORY 节点显示文件夹图标，PAGE 节点显示文档图标
- 底部固定一个按钮：`[+ 新增页面]`（触发已有的向导弹窗）
- 选中节点高亮，右侧面板切换

## 各区域展开后的详细设计

### 区域 1：基本信息（展开状态）

```
▾ 基本信息 ──────────────────────────────────────── [保存]

  页面名称: [______________]    页面编码: [______________]
  路由:     [______________]    图标:     [______________]
  归属目录: [树形选择器 ▾]       排序号:   [__]
```

- 所有字段 inline 编辑
- 修改后点"保存"提交到 `/meta-config/resource`
- 编码只读（创建后不允许修改，避免关联断裂）

### 区域 2：表与列（展开状态）

```
▾ 表与列 ──────────────────────────────────────────────────────

  ┌─ 主表: CostPinggu (T_COST_PINGGU → V_COST_PINGGU) ────────┐
  │                                                             │
  │  ┌────┬────────┬─────┬──────┬────┬────┬────┬────┬───┐     │
  │  │字段│标题    │类型 │TARGET│显示│编辑│查询│控件│序号│     │
  │  ├────┼────────┼─────┼──────┼────┼────┼────┼────┼───┤     │
  │  │... │(AG Grid inline 编辑)                         │     │
  │  └────────────────────────────────────────────────────┘     │
  │  [从视图重新导入] [新增列] [删除列] [保存列]                  │
  └─────────────────────────────────────────────────────────────┘

  ▸ 从表1: CostMaterial (原辅料) FK:DOCID    (点击展开，同上格式)
  ▸ 从表2: CostPackage (包材) FK:DOCID       (点击展开，同上格式)

  [+ 添加从表]
```

- 主表默认展开，从表默认折叠
- AG Grid 直接 inline 编辑列属性
- "从视图重新导入"会重新拉 Oracle 列并对比更新
- 新增/删除列走现有 API

### 区域 3：页面行为（展开状态）

按组件（主表/从表）分块，每块内罗列所有规则类型：

```
▾ 页面行为 ──────────────────────────────────────────────────────

  ┌─ 主表: CostPinggu ──────────────────────────────────────────┐
  │                                                             │
  │  🔘 按钮                                                    │
  │  [查询] [新增] [删除] [保存] [由BOM生成] [同步价格]  [+添加]  │
  │                                                             │
  │  📋 列覆盖(COLUMN_OVERRIDE): 37列已配置      [可视化编辑]     │
  │  📐 计算规则(CALC): 12条                      [可视化编辑]     │
  │  📊 聚合规则(AGGREGATE): 0条                  [可视化编辑]     │
  │  ✅ 校验规则(VALIDATION): 0条                 [可视化编辑]     │
  │  🔒 可编辑规则(EDITABLE): 0条                 [可视化编辑]     │
  │  🎨 样式规则(GRID_STYLE): 0条                 [可视化编辑]     │
  │  ⚙ Grid选项(GRID_OPTIONS): 范围选择、侧边栏   [可视化编辑]     │
  │  🔍 Lookup: GOODSNAME→costGoods, CUSTOM...   [可视化编辑]     │
  │                                                             │
  └─────────────────────────────────────────────────────────────┘

  ┌─ 从表1: CostMaterial (原辅料) ──────────────────────────────┐
  │  (同上结构)                                                  │
  └─────────────────────────────────────────────────────────────┘

  ▸ 从表2: CostPackage (包材)    (折叠，点击展开)
```

交互说明：
- **按钮**：以标签（Tag）形式展示，`flex-wrap` 布局，单行最多 8 个，超出自动换行。点击标签可弹窗编辑该按钮属性，`[+添加]` 新增按钮
- **列覆盖**：点击"可视化编辑"弹出现有 `ColumnOverrideDialog`
- **计算/聚合/校验**：点击"可视化编辑"弹出现有 `RuleConfigDialog`
- **可编辑规则**：点击弹出现有 `CellEditableDialog`
- **样式规则**：点击弹出现有 `GridStyleDialog`
- **Grid 选项**：点击弹出现有 `GridOptionsDialog`
- **Lookup**：点击弹出现有 Lookup 编辑弹窗
- 所有弹窗的 `pageCode + componentKey` 由面板上下文自动传入，用户不需要手动填

### 区域 4：导出配置（展开状态）

```
▾ 导出配置 ──────────────────────────────────────────────────────

  ┌───────────────────┬────────┬──────┐
  │ 模板名称          │ 列数   │ 操作 │
  ├───────────────────┼────────┼──────┤
  │ 标准导出          │ 15列   │ [编辑] [删除] │
  │ 简版导出          │ 8列    │ [编辑] [删除] │
  └───────────────────┴────────┴──────┘

  [+ 新增导出模板]
```

- 点击"编辑"弹出导出配置详情弹窗（配置导出列）
- 数据来源：按当前 pageCode 过滤 `T_COST_EXPORT_CONFIG`

### 区域 5：审批流（展开状态）

```
▾ 审批流 ────────────────────────────────────────────────────────

  ┌───────────────────┬──────────┬────────┬──────┐
  │ 流程名称          │ 节点数    │ 状态   │ 操作 │
  ├───────────────────┼──────────┼────────┼──────┤
  │ 默认审批流         │ 3        │ 启用   │ [编辑] [删除] │
  └───────────────────┴──────────┴────────┴──────┘

  [+ 新增审批流]
```

- 点击"编辑"跳转到审批流配置面板（或弹窗形式展开）
- 数据来源：按当前 pageCode 过滤 `WF_FLOW_DEF`

## 与现有系统的关系

| 项目 | 处理方式 |
|------|---------|
| 现有 6 个 Tab 页 | **保留**，作为"高级模式"入口（批量操作、跨页面查看） |
| 一体化面板 | 新增为**默认视图**，替代现有 Tab 作为首屏 |
| 现有弹窗组件 | **全部复用**（ColumnOverrideDialog、ButtonConfigDialog、GridStyleDialog、RuleConfigDialog、CellEditableDialog、GridOptionsDialog） |
| API 接口 | **全部复用**，不需要新增后端接口 |
| 向导功能 | 保留，由左下角"+ 新增页面"触发 |

## 数据流

```
用户点击左侧树节点
    │
    ▼
读取 pageCode
    │
    ├─→ GET /meta-config/resources (树)
    ├─→ GET /meta-config/tables-by-page?pageCode=xxx (关联表)
    ├─→ GET /meta-config/components (组件 + 按钮配置)
    ├─→ GET /meta-config/rules?pageCode=xxx&componentKey=xxx (各规则)
    ├─→ GET /meta-config/export-configs (导出配置)
    └─→ GET /meta-config/approval/flows (审批流)
    │
    ▼
右侧面板渲染所有区域（折叠摘要 + 按需展开）
```

所有接口**已存在**，不需要新增后端端点。

## 路由设计

- 新页面路由：`/system/meta-config-unified`（或替换现有 `/system/meta-config`）
- URL 参数：`?pageCode=cost-pinggu`（支持直接链接到某个页面配置）
- 兼容：老的 Tab 配置页保留为 `/system/meta-config-advanced`

## 风险与注意事项

1. 右侧面板一次性加载多个区域的数据，需要**懒加载**（折叠区域展开时才请求数据）
2. 拖拽排序需要实时保存 `sortOrder`，避免用户关闭后丢失
3. 弹窗编辑保存后需要**刷新对应区域的摘要信息**
4. 大页面（列数多、规则复杂）展开所有区域时 DOM 较多，AG Grid 应按需渲染
5. 现有 Tab 页保留为高级入口，两套 UI 操作同一份数据，需要注意缓存一致性
